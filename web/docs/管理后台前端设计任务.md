# 管理后台前端开发任务清单

> **项目技术栈**：Vue 3 + TypeScript + Vite + Element Plus + Tailwind CSS + Pinia + Vue Router
>
> **设计目标**：现代商务科技感、高颜值、流畅交互、响应式布局
>
> **任务组织原则**：按照 基础架构 → 认证布局 → 通用业务 → 管理业务 → 细节打磨 的顺序逐步实现

---

## 📋 任务总览

- **阶段一**：项目初始化与基础架构（任务 1-4）
- **阶段二**：认证与布局系统（任务 5-7）
- **阶段三**：通用业务模块（任务 8-9）
- **阶段四**：管理员核心业务（任务 10-12）
- **阶段五**：应用管理与扩展（任务 13-14）
- **阶段六**：UI/UX 极致优化（任务 15-17）

---

## 阶段一：项目初始化与基础架构

### ✅ 任务 1：项目脚手架与配置

**目标**：构建稳健的开发环境

- [ ] 使用 Vite 初始化 Vue 3 + TypeScript 项目
- [ ] 配置 `tsconfig.json`（路径别名 `@/*`）
- [ ] 安装并配置 Tailwind CSS（配置颜色系统、字体）
- [ ] 安装 Element Plus 并配置按需自动导入
- [ ] 配置 ESLint + Prettier 代码规范
- [ ] 配置环境变量 (`.env.development`, `.env.production`)

### ✅ 任务 2：核心依赖库集成

**目标**：集成状态管理与路由

- [ ] 安装并配置 Pinia（状态持久化插件 `pinia-plugin-persistedstate`）
- [ ] 安装并配置 Vue Router（基础路由表）
- [ ] 安装 Axios 并封装 HTTP 请求工具（拦截器、Token注入、统一错误处理）
- [ ] 安装图标库（Element Plus Icons 或其他 SVG 方案）

### ✅ 任务 3：UI 设计系统搭建

**目标**：确立“现代商务科技感”的视觉基调

- [ ] 定义 Tailwind 颜色配置（主色调、强调色、背景色）
- [ ] 封装全局 CSS 变量与基础样式（`index.css`）
- [ ] 实现 **Glassmorphism（毛玻璃）** 工具类或组件
- [ ] 封装通用 UI 组件：
  - `BaseCard` (带阴影、圆角、毛玻璃效果的卡片)
  - `BaseButton` (带微交互动画的按钮)

### ✅ 任务 4：Mock 数据与接口定义

**目标**：完成前后端接口对接准备

- [ ] 根据后端 API 文档，定义 TypeScript 接口类型 (`src/types/*.ts`)
- [ ] 封装 API 请求函数 (`src/api/*.ts`)
  - Auth API
  - User API
  - Card API
  - App API
  - Stats API

---

## 阶段二：认证与布局系统

### ✅ 任务 5：登录页面开发

**目标**：实现高颜值、安全的登录流程

- [ ] 开发登录页 UI（居中卡片、科技感背景、动态效果）
- [ ] 实现登录表单验证（用户名、密码）
- [ ] 集成登录 API（调用 `POST /api/v1/auth/login`）
- [ ] 实现 Token 存储与 Pinia 用户状态管理
- [ ] 实现登录 loading 状态与错误提示

### ✅ 任务 6：整体布局（Layout）开发

**目标**：实现响应式、可扩展的后台框架

- [ ] 开发侧边栏 `SideMenu`（支持折叠、多级菜单、根据角色动态显示）
- [ ] 开发顶部栏 `TopBar`（面包屑、用户信息、个人中心入口、退出登录）
- [ ] 开发主内容区 `AppMain`（`router-view` 过渡动画）
- [ ] 实现路由权限守卫（未登录拦截、无权限跳转 `/forbidden`）

### ✅ 任务 7：动态菜单与权限与路由

**目标**：根据角色动态生成菜单

- [ ] 定义静态路由表（Admin 路由、User 路由）
- [ ] 在 Pinia 中实现 `PermissionStore`
- [ ] 根据 `user.role` 动态生成侧边栏菜单结构
- [ ] 实现 `v-permission` 指令（按钮级权限控制，可选）

---

## 阶段三：通用业务模块（所有用户）

### ✅ 任务 8：统计仪表盘（Dashboard）

**目标**：直观展示核心数据

- [ ] 开发统计卡片组件（数值、趋势、图标）
- [ ] 调用 `GET /api/v1/admin/statistics` 获取数据
- [ ] 根据角色展示不同维度的数据（管理员看全量，用户看个人相关）
- [ ] 图表可视化集成（可选：ECharts 展示趋势图）

### ✅ 任务 9：个人中心（Personal Center）

**目标**：用户自我管理与卡密查看

- [ ] 开发用户信息卡片（头像、基本资料、注册时间）
- [ ] 开发“我的卡密”模块
  - 调用 `GET /api/v1/card/my` 展示卡密信息
  - 实现卡密绑定功能（调用 `POST /api/v1/card/bind`）
  - 实现设备解绑功能（调用 `POST /api/v1/card/unbind-device`）

## 阶段四：管理员核心业务（Admin Only）

### ✅ 任务 10：卡密管理模块

- [ ] 开发卡密列表页（高级筛选、分页、状态标签颜色区分）

### ✅ 任务 11：用户管理模块

**目标**：管控平台用户

- [ ] 状态列高亮显示（正常-绿色，封禁-红色）

### ✅ 任务 12：设备管理模块

**目标**：监控与管理在线设备

- [ ] 开发设备列表页（关联卡密、用户、最后活跃时间）
- [ ] 实现设备强制下线/禁用功能（PUT 请求）

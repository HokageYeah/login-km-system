# 管理后台整体布局与交互设计说明

> 本文档描述 **通用卡密与授权系统管理后台** 的整体 UI 布局、交互方式与可扩展设计。
>
> 目标：
>
> - **现代、美观、专业** 的视觉体验
> - 清晰、高效、偏工具型后台
> - 支持复杂功能但不显臃肿
> - 为未来功能扩展预留空间

---

## 一、整体布局总览

管理后台采用 **经典左右分栏 + 顶部固定栏** 布局，这是中后台系统中**学习成本最低、扩展性最强**的方案。

### 布局结构示意

```
┌──────────────────────────────────────┐
│ TopBar（固定）                        │
│ 卡密系统管理后台   用户信息 / 退出     │
├───────────────┬──────────────────────┤
│ SideMenu      │ MainView              │
│ （左侧菜单）   │ （路由内容区）        │
│               │                       │
│ 一级 / 二级    │ 具体页面              │
└───────────────┴──────────────────────┘
```

---

## 二、顶部栏（TopBar）设计

### 1️⃣ 位置与行为

- **固定在顶部**（`position: fixed`）
- 高度建议：`56px ~ 64px`
- 所有页面共享

---

### 2️⃣ 展示内容

**左侧**：

- 系统名称（固定文案）
  - `卡密系统管理后台`

- 可选：当前环境标识（dev / prod）

**右侧**：

- 当前登录用户名称
- 用户角色标识（Admin / User）
- 下拉菜单：
  - （已实现）个人中心
  - 退出登录

---

### 3️⃣ 设计原则

- 永远可见，强化“系统感”
- 不承载复杂操作
- 仅用于全局信息和会话控制

---

## 三、左侧菜单栏（Side Menu）设计

### 1️⃣ 菜单形态

- 左侧垂直菜单
- **支持两级结构**
- 一级菜单直接展示
- 二级菜单通过点击一级菜单展开

> 展开效果类似「抽屉 / 手风琴（Accordion）」

---

### 2️⃣ 菜单展开交互

- 点击一级菜单：
  - 若有子菜单 → 展开 / 收起
  - 若无子菜单 → 直接跳转

- 同一时间：
  - **只允许一个一级菜单展开**（推荐）

---

### 3️⃣ 菜单结构示例

```text
- 仪表盘
- 用户管理
  ├─ 用户列表
- 卡密管理
  ├─ 卡密列表
  ├─ 生成卡密
- 应用管理
  ├─ 应用列表
- 统计数据
- 个人中心
```

---

### 4️⃣ 菜单权限控制

菜单仅做 **UI 引导**，是否可访问由路由守卫决定。

- 管理员：显示全部菜单
- 普通用户：显示【统计数据】、【个人中心】

---

### 5️⃣ 折叠能力（扩展）

建议支持：

- 左侧菜单 **整体折叠 / 展开**
- 折叠后仅显示图标

> 为未来“功能很多”的场景预留空间

---

## 四、主内容区（Main View）设计

### 1️⃣ 路由渲染区

- 位于右侧
- 使用 `<router-view />`
- 内容随路由变化

---

### 2️⃣ 页面通用结构建议

```text
┌──────── 页面标题 ────────┐
│ 操作区（按钮 / 搜索）     │
├─────────────────────────┤
│ 表格 / 表单 / 内容区      │
├─────────────────────────┤
│ 分页 / 辅助信息           │
└─────────────────────────┘
```

---

### 3️⃣ 页面标题区

- 标题：当前页面名称
- 副标题（可选）：说明当前功能

---

## 五、两级菜单与路由设计关系

### 1️⃣ 一级菜单

- 通常不对应具体页面
- 用于分类与收纳功能

### 2️⃣ 二级菜单

- 对应具体路由页面
- 点击即路由跳转

---

### 3️⃣ 路由设计示例

```ts
{
  path: '/card',
  meta: { title: '卡密管理', roles: ['admin'] },
  children: [
    {
      path: 'list',
      component: CardList,
      meta: { title: '卡密列表' }
    }
  ]
}
```

---

## 六、响应式与可扩展设计

### 1️⃣ 响应式策略

- 桌面端优先
- 小屏幕：
  - 菜单默认收起
  - 顶部栏保留

---

### 2️⃣ 可扩展性设计点

为未来功能预留：

- 新一级菜单（如：订单 / 财务 / 日志）
- 新二级菜单（如：统计子页面）
- 菜单权限细分
- 菜单图标统一管理

---

## 七、UI 风格与体验原则

### 1️⃣ 风格定位

- **现代商务 / 科技感**
- **精致、高级、有质感**
- 拒绝廉价感，使用微触感反馈
- 适当使用 **Glassmorphism（毛玻璃）** 增加层次感
- 鲜明的强调色，柔和的背景色

### 2️⃣ 视觉增强建议

- **阴影与层次**：使用 Tailwind 的 `shadow-sm`, `shadow-md` 区分层级，避免纯平面。
- **圆角**：统一使用 `rounded-lg` 或 `rounded-xl`，保持柔和。
- **动画**：菜单展开、页面切换、按钮点击添加 `transition-all duration-300` 等过渡效果。
- **背景**：主内容区建议使用浅灰/浅蓝背景，卡片使用纯白背景，形成对比。

---

### 2️⃣ Element Plus 与 Tailwind 分工

- Element Plus：
  - 表格
  - 表单
  - 弹窗

- Tailwind CSS：
  - 布局
  - 间距
  - 对齐

---

## 八、状态与反馈设计

### 1️⃣ 页面状态

- loading
- empty
- error

统一处理，避免页面“空白”。

---

### 2️⃣ 操作反馈

- 成功：Message / Notification
- 危险操作：二次确认（Dialog）

---

## 九、拓展建议（未来）

后续可以无痛加入：

- 全局搜索
- 消息通知中心
- 操作日志页面
- 多管理员协作
- 国际化（i18n）

---

## 十、设计总结

- 顶部：**系统身份 + 会话控制**
- 左侧：**功能导航 + 权限可视化**
- 右侧：**稳定承载复杂业务**

> 该布局设计的目标是：
> **三年后再加功能，也不需要推翻重做。**
